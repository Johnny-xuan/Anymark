# 市场调研工作流

用真实信息验证和完善产品想法。

## 调研目标

```
我们要回答的问题：

1. 这个问题真的存在吗？有多少人有？
2. 现有解决方案是什么？好不好用？
3. 市场有多大？在增长还是萎缩？
4. 竞争对手是谁？他们做得怎么样？
5. 用户愿意付费吗？付多少？
```

## 第一步：问题验证

### 1.1 搜索用户反馈

```
搜索关键词：
- "[问题描述] 怎么办"
- "[问题描述] 解决方案"
- "[问题描述] 推荐"
- "[问题描述] 痛点"

搜索平台：
- 知乎、Reddit、Quora（问答）
- Twitter/X、微博（实时讨论）
- 产品论坛、社区（深度反馈）
- App Store/Play Store 评论（产品反馈）
```

### 1.2 量化问题规模

```

- 相关帖子数量
- 讨论热度

市场信号：
- 有没有人在做？

```

## 第三步：市场规模

### 3.1 TAM/SAM/SOM

```
TAM（总可触达市场）：
- 全球/全国有多少潜在用户？
- 市场总价值是多少？

SAM（可服务市场）：
- 我们能触达的用户有多少？
- 考虑地域、语言、渠道限制

SOM（可获得市场）：
- 短期内我们能获得多少？
- 考虑竞争、资源、能力
```
